# 추가 요구사항
생산라인: 생산 큐는 파일로 저장, 각 생산이 끝나는 시점도 같이 저장한다.
그리고 조회 시점에만 파일을 훝어서, 현재 시간 기준으로 지난건 삭제하면서 주문의 상태와 재고량을 바꾼다.
이때 조회 시점이라는 것은 각 주문의 상태를 조회하는 시점과 생산라인 현황, 재고량 등을 조회하는 시점 모두 포함한다.

생산시 부족분 * 수율 만큼 생산하고 실제로는 수율이 곱해진만큼 살아남아야겠지만, 현재 시스템에서는 모두 살아남는 시료로 구현한다. (따라서 재고가 생길 수 있음)

생산라인의 큐에는 주문 승락 시점의 부족분만 들어가고, 대기큐에 들어가있는거나 생산중인 주문은 생각하지 않는다.
따라서 대기중인것과 상관없이 같은 시료에 생산 이후에 들어온 주문은 먼저 confirmed로 넘어갈 수 있다.

현재의 재고량은 생산중이거나 대기중으로 들어간 시료에 의해 선점된다.
예를 들어 50개만 있는 상태에서 100개 주문이 들어가면 50개가 생산중으로 들어갈텐데,
그 상태에서 100개에 대한 주문이 또 생기면 이미 현재 재고인 50개는 이전 주문에서 사용할 예정이므로 100개에 대한 생산주문이 들어가야한다.
confirmed로 넘어가있는 상태에서도, 재고는 표시되지만, 출고 예정이므로 선점되어있어야한다.
따라서 재고량과 여유분이 구분되어야함


주문은 주문 번호가 있어야하고, ex) ORD-숫자4자리
시료는 시료 이름과 시료 ID가 있어야한다.

# 구현 가이드
파일 관리는 Json format으로 할 것이다.
다음 repository의 내용을 참고하여 구현해야한다.
Json parsing 기능: https://github.com/DellaCelina/DataPersistence-dellacelina-1
 - Json의 parsing 및 file 저장에 대한 기능은 위 repo의 구현을 따와서 구현
User interface 기능: https://github.com/DellaCelina/ConsoleMVC-dellacelina-1
 - MVC 구조로 CLI를 구성할 것이며, 위 repo의 MVC 구조를 따와서 구현
Data 관리 기능: https://github.com/DellaCelina/DataMonitor-dellacelina-1
 - Json을 활용한 Database처럼 동작시켜야하므로 위 repo의 구현을 참고
 Dummy data 생성 기능: https://github.com/DellaCelina/DummyDataGenerator-dellacelina-1
 - test를 위한 dummy json data를 생성하는 repo이며, test 구성시 위 repo 구현을 참고해서 구현

Database는 Json format을 사용하며 DataMonitor를 참고해야한다.
각 table은 스키마(json, 각 필드의 속성, 제약 등등 정의)이 정의되어 저장되어야하며,
dummy data generator는 스키마를 기반으로 random한 dummy를 generation하는 기능으로 구현되어야한다.
